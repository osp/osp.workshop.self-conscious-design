Self-conscious-layout

Within the issue of 'Responsive Design', we propose to explore the ePub-format (a standard for digital publications / books) and its implications for media design. Radically different from a traditional book, but much like HTML, an ePub publication adjusts itself to the capabilties of the reading device and the settings of the user. ePub distuinguishes itself in various ways from the other digital format used for digital publications: PDF (Portable Document Format). PDF and ePub propose a fundamentally different approach towards layout. PDF uses a 'fixed' page / layout, which gives the designer a great amount of control, but leaves less flexibility for the digital object to adapt itself to different (digital) contexts. The ePub format describes the layout (in the spirit of LaTeX and HTML) in a more generous way (flow-based layout), allowing the document to re-adjust itself towards the functionalities and properties of the reading-device.
The ePub format is designed so that it can be opened in text editors to access it's code. As with a web page layout, where the handling of HTML tags is part of the process of the designer, the ePub format allows the designer to edit directly in the code. Dealing with code becomes even a necessity when dealing with the interactive features of eBooks, which are defined in the standard but still little explored. What OSP proposes in the workshop, is to test different paths, using both visual tools and direct interventions into the code to and find out where the limits of the ePub paradigm lie.
ePubs use CSS, just like websites, and can make use of the associated technology of media queries. Media queries allow you to adjust your message towards the properties of the device. It enables you to go a step further than just scaling. In traditional print based design we also deal with scale: a message might appear on a poster or a flyer, or the same text might first be published in a newspaper and then in a book. All these media deal with growth in multiple ways, each medium comes with it's own 'scale': in content, detail and time spent by the viewer / reader.
Accomodating for these differents specificities has traditionally been done by designers again and again for each medium. Now the designer needs to imagine the possibilities beforehand, and create a self-aware system that is able to interpret its surroundings. The self-aware publication can, aware of these different factors, adjust itself towards different media. Can you turn a poster into a book of 200 pages? How do you deal with the different scales of content? And when we have self-aware publications, we can also imagine them to perform information hostile to the users. What happens if a book radically censors itself when it's read on a smartphone? Can we expand this self-awareness (geo-awareness, time-awareness) to re-introduce limits wihthin the limitless of the digtal. Perhaps a sort of DRM gone wild?
